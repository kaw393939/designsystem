## Draft and approval chain

Do not skip this sequence.

1. create unit drafts under `content/drafts/units/`
2. create visual drafts under `content/drafts/visuals/`
3. freeze the first reviewable versions under `content/units/` and `content/visuals/`
4. run accessibility, provenance, public-proof, and outside-reviewer checks
5. approve only the artifacts that can actually support release assembly

## Release assembly prerequisites

These artifacts should exist before route implementation begins.

1. `content/experiences/bseai-program.json`
2. route entries in `content/registry/site-registry.json`
3. `content/releases/bseai-release-1.json`
4. at least one exemplar or lead artifact that passes the outside-reviewer test

## Stop rule

If an artifact does not have a governing plan or brief yet, do not draft it just because the route feels obvious.
