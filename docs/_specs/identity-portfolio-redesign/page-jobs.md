# Identity Portfolio Redesign Page Jobs

## Purpose

Define the route map for the redesign so every page has one dominant job and the core assignment path is shorter than the optional theory path.

## Core student path

These routes are the required path for a student trying to finish the assignment and build a stronger portfolio.

| Proposed route | Dominant job | Student question | Must include | Must not include | Next move |
| --- | --- | --- | --- | --- | --- |
| `/experiences/identity-portfolio/` | Start | What is this and where do I begin? | one promise, one example, one primary CTA | full theory stack, full reference library, deep labs | go to signal page |
| `/experiences/identity-portfolio/signal` | Signal choice | What should my portfolio say first? | audience, need, primary archetype, one-sentence promise | design lineage deep dive, publishing system, source library | go to style page |
| `/experiences/identity-portfolio/style` | Perception and visual direction | What should this feel like on first read? | first-screen examples, visual contrasts, design direction choices | full psychology library, full build loop | go to proof page |
| `/experiences/identity-portfolio/proof` | Trust and evidence | What proof belongs near the promise? | artifact selection, proof-block patterns, persuasion guardrails | archetype lab browser, long source list | go to build page |
| `/experiences/identity-portfolio/build` | Assignment build | What do I make next? | assignment ladder, module start, build sequence, output checklist | full research basis, optional labs inline | go to publish page |
| `/experiences/identity-portfolio/publish` | Deployment and opportunity | How do I put this in the world? | publishing note, weak-tie loop, meetup opener, follow-up script | homepage style theory, full archetype explorer | go to examples or sources |

## Optional student support routes

These routes are for specific stuck states. They should never block the core path.

| Proposed route | Dominant job | Use when | Must include | Must not include |
| --- | --- | --- | --- | --- |
| `/experiences/identity-portfolio/diagnose` | Problem diagnosis | your page feels wrong but you cannot name the problem | page-problem chooser, symptom language, recommended next route | full theory lecture |
| `/experiences/identity-portfolio/examples` | Concrete proof | you need to see better page moves in context | before-and-after examples, annotated improvements, short captions | full course framing |
| `/experiences/identity-portfolio/labs/archetypes` | Archetype comparison | you need to compare signals directly | archetype explorer, vocabulary, visual moves, sample proof choices | assignment ladder |
| `/experiences/identity-portfolio/labs/psychology` | Attention and motivation deep dive | you want to understand why a page move works | page problem, principle, fix, example | route map, broad library |
| `/experiences/identity-portfolio/labs/persuasion` | Trust tactics deep dive | the page feels vague, pushy, or fake | authority, social proof, reciprocity, consistency, examples | whole-site overview |
| `/experiences/identity-portfolio/sources` | Research and references | you want the long version and source trail | reading map, research basis, optional references | core build sequence |
| `/experiences/identity-portfolio/system-map` | System overview | you need the full model as an instructor or maintainer | seven-layer model, flow explanation, route map | assignment-first action sequence |

## Support site routes

These routes remain useful, but they are not part of the main student path and should be presented that way.

| Current route | Dominant job | Keep as | Student-facing rule |
| --- | --- | --- | --- |
| `/recipes` | borrow a working page pattern | support route | point to the closest page pattern and get out of the way |
| `/examples/module` | show a module opener pattern | support route | explain the page pattern, not the full course |
| `/examples/lesson` | show a lesson pattern | support route | pace the reading and end with a next move |
| `/examples/reading-map` | show a reading-map pattern | support route | give a way in and a return path |
| `/layouts` | choose the right layout primitive | support route | solve structure problems, not course problems |
| `/tokens` | explain tone and hierarchy jobs | support route | explain why the page feels off or works well |
| `/process` | show how trust is produced | support route | stay out of the way until receipts are needed |
| `/status` | show done versus pending | support route | keep the line between trust and ambition clear |

## Page-level contract

Every redesigned route must satisfy these conditions:

1. one dominant job
2. one dominant question
3. one primary CTA above or near the first screen
4. one visible proof element or concrete example
5. one obvious next move at the bottom

## Forbidden route states

1. a page that starts as an assignment brief and turns into a theory library
2. a page that introduces more than one optional branch before the main action is visible
3. a page that repeats the same pitch in multiple consecutive sections
4. a page that uses support content to delay the build step
