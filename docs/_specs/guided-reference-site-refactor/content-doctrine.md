# Guided Reference Site Refactor Content Doctrine

## Purpose

Define the curatorial, writing, room-template, evidence-tier, and cognitive-load rules for the refactor so the site feels like a museum-school rather than a content dump.

## Voice doctrine

### Core rule

The guided tour should sound like a clear guide helping someone make a decision.

The browse rooms should sound like well-edited exhibits that still remember the learner is here to understand, compare, or apply something.

### Required voice qualities

- direct
- concrete
- historically grounded
- visually aware
- serious without becoming stiff
- explanatory without sounding like a lecture pile

### Disallowed voice qualities

- atlas brochure language with no next move
- decorative theory dumping
- self-congratulatory curation
- vague mood-setting in place of interpretation
- abstract commentary before the concrete page problem

## Room taxonomy

The refactor should use a small set of room types on purpose.

### 1. Lobby room

Job:

- orient the visitor
- sort by role or need
- explain guided versus browse mode

### 2. Guided lesson room

Job:

- teach one decision in the tour
- produce one decision artifact
- send the visitor to the next step

### 3. Compare room

Job:

- help the visitor see meaningful contrast
- narrow choices instead of multiplying them

### 4. Decision room

Job:

- turn comparison into a selected direction
- record anti-patterns and guardrails

### 5. Evidence room

Job:

- prove why a move works through examples, receipts, annotations, or cited support

### 6. Source room

Job:

- expose deeper research, provenance, and supporting material without taking over the main tour

## Lobby first-screen blueprint

The lobby is an exhibit threshold and a usability threshold.

Its first screen should contain only:

1. one dominant CTA into the guided tour
2. one secondary CTA for returning builders or examples
3. one proof object or example cue
4. one plain-language sentence explaining guided versus browse mode
5. no more than four role or intent options

It should not contain:

1. full compare grids
2. multiple equal-priority browse teasers
3. dense theory framing
4. a flat list of deep rooms above the fold

## Route-status labels

Every route should visibly declare its status.

Allowed statuses:

1. `Entry`
2. `Required in tour`
3. `Recommended support`
4. `Optional reference`
5. `Instructor only`
6. `Wrapper-specific`

## Object hierarchy

The site should stop treating every object as equally important.

### Hero object

The single object that does the main interpretive work of the room.

Examples:

1. one annotated before-and-after page comparison
2. one archetype cluster map
3. one proof-block anatomy board
4. one lineage comparison board

### Supporting object

An object that sharpens, contrasts, or tests the claim of the hero object.

Examples:

1. one secondary example
2. one captioned diagram
3. one short quote rail with clear interpretive job

### Reference object

An object that supports deeper study but is not necessary to understand the room's dominant move.

Reference objects belong lower in the room or in expandable support surfaces.

## Room-template budgets

The budgets below are default caps, not goals to fill.

### Guided lesson room budget

1. one opening interpretation panel
2. one hero object
3. one to two supporting objects
4. zero to one quote rail
5. zero to two evidence cards
6. one explicit decision output
7. one clear next move

### Compare or browse room budget

1. one opening interpretation panel
2. one hero compare surface
3. two to four supporting objects
4. zero to two quote rails
5. zero to two evidence cards
6. one `what this changes` panel
7. one `return to tour` handoff

### Source room budget

1. one framing panel
2. three to six source clusters
3. visible evidence-tier labels for each cluster
4. no requirement to force a build step, but do provide related-room links

## Section and density rules

### Summary-before-depth rule

The visitor should be able to skim the room and understand its main move before deciding to read further.

Every room should present in this order:

1. what this room is for
2. the main object or contrast
3. what this changes
4. deeper study or supporting material

### One high-entropy surface per screen rule

Do not stack multiple dense diagrams, compare boards, long timelines, or large grids on the same screen without an interpretive break.

### Short-section preference

Prefer several bounded sections with visible labels over one massive continuous prose block.

## Quote-rail rules

Quotes should sharpen interpretation, not create ambience.

Allowed quote jobs:

1. name the governing idea of the room
2. expose a contradiction or trap
3. connect the room to a verified source anchor

Rules:

1. no more than two quote rails in a standard room
2. every quote needs a caption or context note
3. do not use quotes when a direct explanation would do the job better

## Evidence tiers and provenance labels

Every room should visibly distinguish the status of its objects.

### `Verified evidence`

Use for:

1. cited research findings
2. verified historical facts
3. real screenshots, artifacts, or documented outcomes

### `Interpreted synthesis`

Use for:

1. concise explanation built from multiple sources
2. curator-authored takeaways that compress a research trail

### `Concept model`

Use for:

1. diagrams and boards that simplify a teaching model
2. abstractions such as ladders, chains, matrices, and maps

### `Generated illustration`

Use for:

1. image-based atmosphere or metaphor
2. illustrative scenes that are not factual evidence

Rules:

1. generated illustration must never be mistaken for documentary evidence
2. concept models must say what they compress and what they leave out
3. interpreted synthesis should point back to the source room when the room is making a bigger claim

## Object-label schema

Every hero object and every supporting object should carry a compact label or caption package.

Required fields when relevant:

1. object title
2. object type
3. date or period
4. source or provenance
5. why this object is here
6. what this object proves or changes
7. evidence tier

This applies especially to:

1. portraits and historical figures
2. screenshots and page fragments
3. diagrams and compare boards
4. charts and other factual graphics

## Internal navigation and CTA rules

### Guided pages

1. one primary forward CTA
2. one bounded secondary CTA into optional depth
3. next move appears above the fold or near the end of the main section sequence

### Browse rooms

1. one explicit `return to the tour` CTA
2. one explicit `compare next` or `related room` CTA
3. one panel explaining what build decision this room should influence

## Tour-record display rules

Every guided room should display the current learner record in a compact panel or sidebar.

At minimum the visitor should be able to see:

1. what has already been decided
2. what this room is deciding now
3. what field will be added or changed after this room

Browse rooms should point to the exact learner-record field they are helping sharpen.

## Misconception and transfer blocks

Every guided room should include:

1. one `common mistake` or `trap` block
2. one `apply this now` or `transfer to your page` block

The purpose is not only to explain the concept, but to help the learner use it immediately.

## Formative-check rules

Every guided room should end with one small check that can be completed quickly.

Good checks are:

1. observable
2. fast
3. tied to the room's single job
4. usable by both a solo learner and a peer reviewer

## Multimedia rules

Multimedia is part of the teaching method, not decoration.

### Required media package rules

1. audio requires a transcript
2. charts require a concise takeaway sentence
3. charts and diagrams that need it require a long description
4. annotated visuals should explain the decision they are meant to support
5. no auto-playing or attention-stealing motion on deep rooms

## Cognitive-load guardrails

1. keep the first screen of any room sparse enough to scan quickly
2. do not require the visitor to understand more than one major framework before the first practical takeaway appears
3. use cards, captions, and labels to chunk meaning, not just to add ornament
4. keep deep source density in source rooms or expandable sections, not at the top of guided pages
5. do not place more than one high-entropy teaching surface before the first apply-now takeaway
6. keep the lobby and guided pages visually simpler than deep browse rooms

## Forbidden states

1. a room that feels rich but leaves the visitor unsure what changed
2. a dense room with no hero object and no hierarchy
3. a quote-led room where the quotes do more work than the interpretation
4. a room that uses generated imagery as if it were proof
5. a guided page that behaves like a source library
6. a browse room with no explicit path back to action
7. a guided page with no misconception block and no formative check
