# BSEAI IA Review Synthesis and Action Plan

## Status

- Status: synthesis of the expert-lens review set
- Scope: consolidated changes to apply to the IA, wrapper, and media planning docs

## Shared conclusions

Across the review set, the same issues recur.

### 1. The IA needs stronger author-facing build rules

The public architecture is clear.

The authoring architecture still needs:

1. object placement rules
2. release-1 visibility rules
3. asset state and asset spec rules

### 2. The atlas needs tighter instructional and curatorial boundaries

The atlas is a real strength, but it needs:

1. room templates
2. required-versus-optional wrapper use
3. theory-to-action handoff rules
4. explicit limits on room density

### 3. The archetype system needs a decision rubric, not only a structure

The Hero and the Outlaw integration improved the atlas significantly.

It still needs:

1. a primary-archetype selection rubric
2. a congruence checklist
3. a category-essence note

### 4. The media system needs stronger production metadata

The route-by-route media plan is good, but it needs:

1. asset specs
2. chart-card template
3. script package template
4. audio-selection rubric
5. media QA gates

### 5. Accessibility and provenance need to move closer to the center of the IA

The system now assumes source rigor and accessibility, but the review set says those rules should be explicit enough to govern production.

That means:

1. evidence tiers
2. quote verification rules
3. synthetic versus evidence labeling
4. transcript and long-description rules
5. cognitive-load guardrails

## Action set to apply now

### IA changes

1. add an IA object model and placement table
2. add release-1 visibility rules
3. add a room template and density guidance
4. add required-versus-optional atlas rules
5. add archetype selection rubric and congruence checklist
6. add evidence tiers and provenance rules
7. add a first-time visitor CTA ladder

### IS117 changes

1. add atlas usage status by phase
2. add strategy-pack crosswalk from gallery outputs to assignment fields
3. add theory-to-action guardrails for early weeks
4. add outside-reviewer and public-proof checks

### Media plan changes

1. add asset-spec template
2. add chart-card template
3. add script package template
4. add audio-selection rubric
5. add asset state model
6. add media QA gates
7. add accessibility constraints for charts, rooms, and audio

## Order of implementation

1. patch the parent IA first because it governs the rest
2. patch the IS117 wrapper second because it operationalizes the IA
3. patch the media plan third because it converts the IA into assets
4. update the project indexes last so the new review layer is findable

## Success condition

The synthesis is successful when:

1. the IA becomes easier to author from
2. the atlas becomes easier to curate and teach from
3. the archetype system becomes harder to misuse
4. the media plan becomes specific enough to hand to production
5. source rigor and accessibility become build rules rather than reminders
