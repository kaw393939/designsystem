# Instructional Design and Curriculum Review

## Status

- Lens: instructional designer and curriculum architect
- Review target: BSEAI site IA and IS117 wrapper

## Findings

1. The IA has a strong module shell and outcomes map, but atlas rooms are not yet clearly marked as required, optional, or wrapper-specific learning experiences.
2. The gallery outputs are named, but the handoff from gallery decision artifacts into the IS117 strategy pack and later build milestones should be more explicit.
3. The module sequence is good, but the dependency chain between orientation, why-now, signal, archetype, style, proof, build, and publish is not expressed as a prerequisite map.

## Risks

1. Instructors may over-assign atlas content because its status in the wrapper is not explicit.
2. Students may complete gallery activities without understanding how those decisions must show up in the next assignment.
3. Later course wrappers may sequence canonical modules incorrectly because dependencies are inferred rather than stated.

## Required changes

1. Add atlas usage rules that distinguish `required in wrapper`, `recommended support`, and `optional depth`.
2. Add a strategy-pack crosswalk showing exactly which gallery decisions feed which assignment fields.
3. Add a simple prerequisite map for Phase 1 modules so future wrappers preserve the intended learning logic.
