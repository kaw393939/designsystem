# Psychology, Attention, and Persuasion Gallery

## Status

- Status: active feature brief
- Product role: sibling gallery to archetype and visual-design literacy experiences
- Teaching goal: explain why design moves work, what they optimize for, and where the ethical boundaries are

## Purpose

Build a reusable gallery that helps students and product designers understand the psychology behind:

1. attention
2. scanning
3. first-read interpretation
4. trust formation
5. persuasion and commitment
6. ethical versus manipulative influence

## Why this matters

The site already uses ideas about gaze, hierarchy, proof, and persuasion.
Students need to understand those moves as structured design choices, not myths, hacks, or vibes.

## Working doctrine

This gallery should teach psychology through page problems.

Not:

1. a giant theory library
2. a list of tricks
3. a pop-psych glossary

Instead it should answer:

1. what is happening on the page
2. why users are reacting that way
3. what to change
4. what misuse looks like

### 6. Persuasion structure

## Gallery modes

### 1. Browse mode

Students browse principles as cards tied to specific page problems.

### 2. Diagnose mode

Students start from a failure state:

1. nobody notices the headline
2. the page feels noisy
3. the page feels pushy
4. users do not trust the claim
5. the CTA is ignored

### 3. Experiment mode

Students see testable comparisons:

1. face versus no face
2. proof above fold versus below fold
3. grid-first versus interrupt-first
4. ad-like panel versus integrated content block

### 4. Ethics mode

Students learn where influence stops being help and starts becoming manipulation.

## Data model for each principle card

Each card should include:

1. page problem
2. principle name
3. concise evidence-backed explanation
4. what users do
5. design move
6. common misuse
7. ethical guardrail
8. metric or observation to test
