# Projects Index

## Purpose

This folder is the working project layer for active builds that cut across research, curriculum, product design, and site execution.

Use it when the work is no longer only:

- source research in `_research/`
- synthesis in `_content/`
- durable policy in `_specs/`
- UX evidence in `_ux/`

Project docs should translate those layers into concrete builds, sequencing, decisions, and open questions.

## Current projects

- `identity-first-site/`: the first student-facing site for the first course, centered on the identity portfolio system, archetype exploration, and real-world deployment
- `bseaid-degree/`: degree positioning, content-production doctrine, route planning, course-spine mapping, archive guidance, and program-shaping doctrine for the BSEAID direction; start with `bseaid-degree/README.md`
- `research-backlog.md`: shared open questions and next research passes feeding both projects

## Use rules

1. Keep project docs action-oriented.
2. Link back to research and spec docs instead of duplicating them wholesale.
3. Record assumptions, decisions, risks, and next steps explicitly.
4. Promote stable rules into `_specs/`, `_research/`, or `_content/` once they stop being project-specific.
5. Do not let this folder turn into a second archive of loose notes.
