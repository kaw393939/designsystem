# Archetype Imagery Audit

This pass audits the public workshop routes for where archetypal and proof-oriented imagery should do more work. The goal is not decoration. The goal is to make the site visibly practice the same system it teaches: signal, style, proof, and public-facing coherence.

## Public Route Families

- `/` `Start`: Implemented a new orientation board. The homepage now previews audience, vibe, look, and proof instead of only describing the route families in text.
- `/tour` `Tour`: Implemented a six-step storyboard. The landing page now reads more like a build sequence and less like a syllabus overview.
- `/browse` `Browse`: Implemented a comparison-room board. The section now frames browse as a wall of decisions instead of a pile of extra reading.
- `/examples` `Examples`: Implemented an outcome-proof spread. The landing page now shows finished outcomes and receipts together.

## Guided Steps

- `/tour/signal`: Implemented an audience-brief board. This step now shows the kind of human-centered input the system expects before style decisions begin.
- `/tour/archetype`: Implemented a portrait compare board and image-backed candidate cards using the atlas archetype portraits. This step now applies archetypal imagery directly to the comparison task.
- `/tour/style`: Already strong. Existing style boards are doing real work and should remain the primary imagery reference here.
- `/tour/proof`: Implemented a proof-receipt board. This step now models screenshots, testimonial objects, and metrics as a coherent trust system.
- `/tour/build`: Implemented a wireframe-and-review board. This step now shows the page skeleton, proof placement, and critique surface above the fold.
- `/tour/publish`: Implemented a public-release board. This step now frames launch as a visible mix of live-site review, share artifacts, and follow-up signals.

## Browse Rooms

- `/browse/archetypes`: Implemented an above-the-fold compare strip plus image-backed comparison cards for the highlighted archetypes. The room now starts with visible family contrast before the concept grid and deep references.
- `/browse/design-lineages`: Implemented a hero strip that previews the main visual lanes before the deeper style boards. The room now shows the clarity-versus-friction range above the fold instead of only in-content.
- `/browse/attention-trust`: Implemented a hero strip that contrasts authority, reciprocity, and social proof before the deeper persuasion cards. The room now starts with the three most useful trust levers instead of only the longer explanations below.
- `/browse/sources`: Implemented a provenance-map hero board. The room now makes the research trail visible before the source clusters and evidence tiers take over.

## Example and Support Routes

- `/examples/student-exemplars`: Implemented an end-to-end exemplar spread plus image-backed portfolio and museum exemplar boards. The route now shows finished outcomes and decision-trail artifacts together.
- `/examples/proof-blocks`: Implemented a dedicated before-and-after proof montage and kept the compare cards beneath it as supporting annotation. The route now shows the visual change before the anatomy notes begin.
- `/recipes`, `/layouts`, `/tokens`, `/process`, `/status`, `/primitives`: These already use custom system visuals or diagrammatic illustrations. They are in better shape than the top-level public route-family pages.

## Remaining High-Value Passes

- `/tour/style`: The existing boards are still strong, but a tighter hero-level compare strip could make the step read faster on first arrival.
- `/browse`: The browse landing already has a room-level board, but it could eventually use smaller inline previews for each canonical room.
- `/examples`: The family landing could eventually preview proof-block and exemplar imagery directly inside the route cards rather than relying on text-first cards.

## Design Rule

Use imagery when it clarifies the decision the page is asking the student to make. If an image does not help with signal, comparison, proof, or transfer, it is probably decorative noise.
