## Master line

Capture attention with tension, build trust with clarity, and convert through structured action, then show up in public until opportunity compounds.

## Master model

Motivation -> Identity -> Perception -> Trust -> Action -> Deployment -> Opportunity

That is the whole machine.

## Layer jobs

### 1. Motivation

Start with human need, not features.

- Safety: help people feel less lost in a changing market
- Belonging: help people find a group or field they can enter
- Esteem: help people feel credible, respected, or ahead
- Self-actualization: help people become more fully themselves through what they build

Rule: use Maslow as a map, not a ladder.

### 2. Identity

People read coherent identities faster than mixed signals.

- Choose one primary archetype
- Optional: one subtle secondary archetype
- Never let the first read feel like three different personalities collided

Output: voice, promise, tone, and proof standard.

### 3. Perception

Perception decides what the page feels like before the reader has processed the argument.

- Swiss logic: calm hierarchy, legibility, trust
- Brutalist logic: interruption, tension, edge

Core rule: attention can interrupt the scroll, but clarity must take over quickly.

### 4. Trust

Trust is not mood. Trust is structure.

- Put proof near the promise
- Use authority as real method and real artifact
- Use social proof as visible response or reuse
- Use reciprocity as genuine help before the ask

Rule: persuasion must reduce uncertainty honestly.

### 5. Action

Every page needs a next move.

The useful sequence is:

1. Attention
2. Tension
3. Clarity
4. Authority
5. Action

If the action arrives before clarity and trust, it feels needy. If it never arrives, the page stays decorative.

### 6. Deployment

The portfolio is not finished when the URL works.

Deployment means the same signal leaves the browser and enters:

- posts
- comments
- meetup conversations
- follow-up notes
- short talks

Weak ties matter because they often create the first opening.

### 7. Opportunity

Opportunity comes from repeated exposure plus visible competence.

Signals of traction include:

- replies
- interviews
- freelance asks
- collaborator intros
- talk invitations

The page does not have to be perfect. It has to be clear enough to travel.

## Student translation

If the full model feels abstract, teach the shorter version:

1. Need
2. Signal
3. Style
4. Proof
5. CTA
6. Publish
7. Repeat

## Invariants

1. One real audience at a time.
2. One primary archetype at a time.
3. One clear promise on the first screen.
4. Proof must sit close to the claim.
5. Every page needs a next step.
6. Deployment is part of the system, not a bonus round.

## Forbidden states

1. Mixed archetypes with no hierarchy.
2. Decorative visuals standing in for strategy.
3. A polished page that still says nothing.
4. A CTA with no trust behind it.
5. A finished page that never enters public circulation.

## Deployment ladder

Use the smallest real-world loop first.

1. Publish the page or proof block.
2. Share it in one public channel.
3. Bring it into one live conversation.
4. Follow up with the link.
5. Note what people remembered first.
6. Tighten the next version.
