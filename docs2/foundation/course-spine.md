# BSEAID Eight-Course Spine

## Status

- Status: working curriculum architecture note
- Source catalog sheet: `docs/_research/bseai.md`

## Purpose

Treat the eight-course IS spine as the narrative backbone of the degree.

The point is not only to list required courses.
The point is to give each spine course a visible role in forming an AI-native builder.

## Working rule

This is a program-shaping sequence, not a registrar-only view.
Some courses may currently appear elsewhere in the official catalog order.
This document treats them as the eight-course story the degree tells.

## Proposed spine sequence

| Spine slot | Course | Degree job | Signature artifact |
| --- | --- | --- | --- |
| 1 | IS117 | teach public page construction and first web fluency | first portfolio page and signal surface |
| 2 | IS265 | introduce information systems, actors, workflows, and organizational context | systems map and problem brief |
| 3 | IS218 | turn the page into an interactive product and application workflow | working web application with user-facing flows |
| 4 | IS331 | ground students in schema, data structure, retrieval, and information architecture | data model and retrieval-aware application layer |
| 5 | IS390 | teach discovery, requirements, stakeholder translation, and system framing | requirements package and product definition |
| 6 | IS425 | specialize in enterprise AI applications and infrastructure | LLM or agentic system with evaluation and deployment notes |
| 7 | IS455 | connect systems to business process, adoption, and operational change | workflow redesign and automation plan |
| 8 | selected IS course | deepen the program's chosen edge | advanced artifact aligned to product, orchestration, or governance |

## Supporting courses outside the spine

These courses should reinforce the spine even if they are not part of the eight-course story:

1. DS110 for AI foundations
2. CS118 for programming depth
3. IS247, IS375, and IS448 for UX research and usability
4. IS392 and IS480 for AI and data depth
5. MGMT 480 and IE 492 or ENTR 210 for innovation and organizational framing
6. IS491 or IT 491 for capstone execution

## What each spine course should answer

| Course | Core question |
| --- | --- |
| IS117 | Can you make yourself legible on the web |
| IS265 | Do you understand the systems and institutions your product must live inside |
| IS218 | Can you build a real interactive product |
| IS331 | Can you structure and retrieve information coherently |
| IS390 | Can you translate ambiguity into system requirements |
| IS425 | Can you build and operate enterprise AI infrastructure responsibly |
| IS455 | Can you connect technical systems to real business process change |
| selected IS course | What is your distinctive depth |

## Candidate advanced eighth-course directions

Pick the final IS course based on the identity the degree wants to emphasize.

### If the degree leans product and deployment

- stronger UX and product-research option

### If the degree leans LLM systems and orchestration

- stronger AI, data, or platform-oriented option

### If the degree leans governance and enterprise change

- stronger policy, measurement, or organizational systems option

## Release gate for the eighth slot

Until the program leadership group chooses the final course, public-facing planning should treat slot 8 as selected advanced depth rather than a settled catalog promise.

Use these criteria for the decision:

1. the course strengthens the degree's chosen edge rather than duplicating IS425 or IS455
2. the course ends in an externally legible artifact, not only internal analysis
3. the course helps the student deepen product, orchestration, or governance capability in a way employers can understand
4. the course can be staffed and defended repeatedly as part of the degree identity
5. the course completes the artifact ladder rather than breaking it

Working rule:

1. do not publish a fixed course code or title for slot 8 until the choice is approved
2. keep the course-spine overview and visual ladder honest about the placeholder status
3. lock the decision before `bseai-course-spine-overview` is treated as approved public-facing copy

## Program design implication

Every spine course should end with a public or externally legible artifact.
The portfolio system then becomes the place where the entire spine becomes visible over time.
